**Feature**: `006-youtube-processor` | **Date**: 2026-04-08

## Entities

### ParsedYouTubeUrl

Canonical identity for a user-supplied YouTube URL.

| Field              | Type                                           | Description                                       |
| ------------------ | ---------------------------------------------- | ------------------------------------------------- |
| `originalUrl`      | `string`                                       | URL pasted by the user or discovered in context   |
| `canonicalUrl`     | `string`                                       | Normalized watch URL used for display and storage |
| `videoId`          | `string`                                       | Stable 11-character YouTube ID                    |
| `startTimeSeconds` | `number \| undefined`                          | Parsed start offset from query/path parameters    |
| `sourceKind`       | `"watch" \| "short" \| "embed" \| "shortlink"` | Which URL form was matched                        |

### YouTubeVideo

Metadata about a processed YouTube video.

| Field                | Type                  | Description                               |
| -------------------- | --------------------- | ----------------------------------------- |
| `videoId`            | `string`              | Canonical YouTube video ID                |
| `url`                | `string`              | Canonical video URL                       |
| `title`              | `string`              | Video title                               |
| `channelName`        | `string \| undefined` | Channel or author name                    |
| `description`        | `string \| undefined` | Video description when available          |
| `publicationDate`    | `string \| undefined` | ISO-like publication/upload date          |
| `durationSeconds`    | `number \| undefined` | Duration in seconds when known            |
| `thumbnailUrl`       | `string \| undefined` | Thumbnail URL if provider exposes one     |
| `availableLanguages` | `string[]`            | Languages reported by the provider        |
| `chapters`           | `VideoChapter[]`      | Parsed chapter list derived from metadata |

### VideoChapter

| Field              | Type                  | Description                                        |
| ------------------ | --------------------- | -------------------------------------------------- |

### TranscriptSegment

Normalized transcript unit shared by all providers.

| Field              | Type                  | Description                       |
| ------------------ | --------------------- | --------------------------------- |
| `text`             | `string`              | Spoken text for the segment       |
| `startTimeSeconds` | `number`              | Segment start offset              |
| `endTimeSeconds`   | `number \| undefined` | Segment end offset if known       |
| `durationSeconds`  | `number \| undefined` | Duration when provider returns it |
| `speaker`          | `string \| undefined` | Speaker label if ever available   |

### VideoTranscript

Provider-agnostic transcript payload used by the formatter and context system.

| Field               | Type                                   | Description                                              |
| ------------------- | -------------------------------------- | -------------------------------------------------------- |
| `videoId`           | `string`                               | Source video reference                                   |
| `provider`          | `string`                               | Caption or transcription provider identifier             |
| `language`          | `string`                               | Returned transcript language code                        |
| `extractionMethod`  | `"captions" \| "audio" \| "webviewer"` | How the transcript was obtained                          |
| `isAutoGenerated`   | `boolean`                              | Whether captions were machine-generated                  |
| `segments`          | `TranscriptSegment[]`                  | Ordered normalized transcript segments                   |
| `plainText`         | `string`                               | Flattened transcript text for search and quick summaries |
| `formattedMarkdown` | `string`                               | Timestamped transcript with optional chapter headings    |
| `wordCount`         | `number`                               | Aggregate word count                                     |
| `tokenEstimate`     | `number \| undefined`                  | Rough token estimate for compaction decisions            |
| `extractedAt`       | `number`                               | Epoch ms when the transcript was fetched                 |

| Field        | Type              | Description                                                    |
| ------------ | ----------------- | -------------------------------------------------------------- |

### SavedTranscriptNote

Represents the persisted vault artifact for a transcript export.

| Field             | Type      | Description                                                   |

## Validation Rules

1. `videoId` must match the canonical 11-character YouTube ID pattern.
2. `canonicalUrl` must always resolve to a watch-style HTTPS URL.
3. `startTimeSeconds`, `durationSeconds`, and chapter boundaries must be `>= 0`.
4. Transcript segments must be stored in ascending `startTimeSeconds` order.
5. `formattedMarkdown` must be derived from `segments`, not treated as a source of truth.
6. Cache hits are valid only when `expiresAt > Date.now()` and `version` matches the current formatter schema.
7. Saved transcript notes must include at minimum the title, source URL, and transcript body.

## State Transitions

### Transcript Processing Lifecycle

```text
User URL
  -> parse and normalize URL
  -> check cache
      -> cache hit -> return cached transcript
      -> cache miss -> fetch video metadata + captions
          -> captions found -> normalize segments -> format markdown -> cache -> inject into context
          -> captions missing -> attempt configured audio fallback
              -> fallback success -> normalize -> format -> cache -> inject into context
              -> fallback failure -> return structured error
```

### Save-to-Vault Lifecycle

```text
Processed transcript
  -> build note content
  -> resolve output folder
  -> detect existing note collision
      -> overwrite path chosen
      -> or disambiguated file name chosen
  -> persist markdown note
```

## Access Patterns

| Operation                 | Frequency                                     | Method                                          |
| ------------------------- | --------------------------------------------- | ----------------------------------------------- |
| Parse YouTube URL         | Every pasted or mentioned YouTube URL         | `parseYouTubeUrl()`                             |
| Extract transcript        | Per tool, mention, or project context request | `YouTubeExtractor.extractTranscript()`          |
| Format transcript         | Once per successful fetch or cache refresh    | `YouTubeTranscriptFormatter.formatTranscript()` |
| Parse chapters            | Once per metadata payload with description    | `parseYouTubeChapters()`                        |
| Check cache               | Every extraction request                      | `YouTubeCache.get()`                            |
| Save transcript note      | User-invoked export path                      | `YouTubeTranscriptNoteWriter.save()`            |
| Inject transcript context | Tool and mention completion                   | `buildYouTubeContextBlock()`                    |
